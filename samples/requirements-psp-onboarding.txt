REQUIREMENTS SPECIFICATION
PSP Onboarding Module - Bank Indonesia Payment Infrastructure
Version 2.1 | March 2026

REQ-001: PSP Registration Portal
The system shall provide a secure web portal for Payment Service Providers to submit
registration applications. Required fields: PSP Name, License Number, Business Type
(bank/non-bank/fintech/e-money), Contact Person, Email, Phone, Address.
Acceptance Criteria: All mandatory fields validated before submission, duplicate
license number check performed in real-time.

REQ-002: Document Upload & Verification
The system shall allow PSPs to upload supporting documents including Business License,
Bank Guarantee Letter, Technical Compliance Certificate, and Latest Financial Statement.
Maximum file size: 25MB per document, supported formats: PDF, DOCX, JPG, PNG.
Acceptance Criteria: File type and size validation, virus scan on upload, document
hash generated for integrity verification.

REQ-003: KYC/AML Screening
The system shall perform automated Know Your Customer and Anti-Money Laundering
screening on all PSP principals, directors, and authorized signatories against
PPATK (Indonesian Financial Transaction Reports and Analysis Centre) watchlists.
Acceptance Criteria: Screening completes within 30 seconds, match confidence
threshold configurable (default 85%), results logged with timestamps.

REQ-004: Technical Readiness Assessment
The system shall evaluate PSP technical infrastructure readiness including:
- API endpoint availability (health check must return 200 within 500ms)
- TLS 1.3 certificate validation
- ISO 8583 message format compliance testing
- BI-FAST settlement file format compliance
- Sandbox environment connectivity test
Acceptance Criteria: All 5 technical checks must pass, detailed report generated.

REQ-005: Risk Score Calculation
The system shall calculate a composite risk score (0-100) for each PSP application
based on weighted criteria: Financial Health (30%), Technical Readiness (25%),
Compliance History (25%), Operational Capacity (20%).
Acceptance Criteria: Score calculated within 5 seconds, breakdown by category
displayed, audit trail of score changes maintained.

REQ-006: Multi-Level Approval Workflow
The system shall route PSP applications through approval levels:
- Level 1: Automated validation (system checks)
- Level 2: Compliance Officer review and sign-off
- Level 3: Technical Committee review (2 of 3 members must approve)
- Level 4: Director approval (required when risk score < 60)
Acceptance Criteria: SLA tracking per level (L1: 1hr, L2: 2 days, L3: 3 days,
L4: 5 days), escalation notifications on SLA breach.

REQ-007: Credential Provisioning
Upon final approval, the system shall automatically provision:
- API credentials (client_id + client_secret, RSA-256 signed)
- Mutual TLS digital certificates (validity: 1 year)
- Sandbox environment access with test data
- Production environment access (unlocked after sandbox certification)
Acceptance Criteria: Credentials generated within 60 seconds, delivered via
encrypted channel, credential rotation policy enforced every 90 days.

REQ-008: Batch Onboarding Support
The system shall support batch processing of up to 25 PSP applications per
quarterly cycle. Batch phases: Submission (Week 1-2), Verification (Week 3-4),
Assessment (Week 5-6), Approval & Provisioning (Week 7-8).
Acceptance Criteria: Batch dashboard shows pipeline status, all 25 PSPs
processed within 8-week cycle, parallel processing supported.

REQ-009: Real-Time Dashboard
The system shall provide dashboards showing:
- Application pipeline (submitted / in-review / approved / rejected)
- Processing time per stage with SLA indicators
- KYC screening results summary
- Quarterly onboarding metrics and trends
- Batch progress tracking
Acceptance Criteria: Dashboard auto-refreshes every 5 minutes, exportable
to PDF/Excel, role-based access (admin sees all, officer sees assigned).

REQ-010: Comprehensive Audit Trail
The system shall maintain immutable audit logs including:
- All user actions with timestamps and IP addresses
- System automated actions with trigger details
- Document access and download logs
- Approval/rejection decisions with rationale text
- Credential generation and rotation events
Acceptance Criteria: Logs searchable by date/user/action, retained 7 years
minimum, tamper-proof with cryptographic hash chain.

REQ-011: Notification System
The system shall send notifications via email and SMS for:
- Application status changes
- Document verification results
- Approval/rejection decisions
- SLA breach warnings (at 80% of SLA time)
- Credential expiry reminders (30, 14, 7 days before)
Acceptance Criteria: Notifications delivered within 2 minutes, configurable
templates, opt-out available for non-critical notifications.

REQ-012: Data Privacy & Security
The system shall comply with Bank Indonesia information security policies:
- All data encrypted at rest (AES-256) and in transit (TLS 1.3)
- PII data masked in logs and non-production environments
- Role-based access control with principle of least privilege
- Session timeout after 15 minutes of inactivity
- Multi-factor authentication for admin users
Acceptance Criteria: Annual security audit passing, zero critical
vulnerabilities in penetration test results.
