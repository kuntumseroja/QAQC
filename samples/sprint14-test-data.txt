7. RISK ASSESSMENT
-------------------

  RISK-01: Bulk Payment Performance (HIGH)
    Description: Bulk payment API does not meet 5000ms P95 SLA target.
    Sequential processing design requires architectural refactor to async queue.
    Impact: BIP-340 cannot be released until resolved.
    Mitigation: Sprint 15 spike to design async bulk processor using Kafka.
    Owner: Dev Lead

  RISK-02: mTLS Performance Overhead (MEDIUM)
    Description: mTLS adds 312ms to P95 handshake time; target is 200ms.
    Adding full PKI validation per-request is expensive at scale.
    Impact: API Gateway may not meet SLA during peak load with mTLS.
    Mitigation: Cache OCSP responses in Redis (24h TTL); session resumption.
    Owner: Security Architect + Dev Team

  RISK-03: SCD Type 1 on PSP Dimension (LOW-MEDIUM)
    Description: PSP tier changes overwrite historical data. Defect #036 from
    Data Pipeline team. Not a Sprint 14 story but surfaced during testing.
    Impact: Historical regulatory reports may be inaccurate.
    Mitigation: Escalated to Data Engineering; Sprint 16 design sprint planned.
    Owner: Data Architect

  RISK-04: log4j Dependency (HIGH)
    Description: log4j-core 2.17.0 included via transitive dependency (Spring Boot).
    OWASP check flags it as critical CVE.
    Impact: Security non-compliance; cannot release to production.
    Mitigation: Update Spring Boot parent to 3.2.x which ships log4j 2.21.0.
    Owner: Dev Lead — Target: Sprint 15 Day 1
