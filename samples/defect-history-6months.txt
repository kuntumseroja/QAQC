DEFECT #002
  ID:            BIPAY-1052
  Title:         Duplicate transaction IDs generated under high-concurrency insert
  Severity:      P1 (Critical)
  Module:        Transaction Processing
  Sub-module:    Transaction ID Generator
  Root Cause:    RACE — UUID generation used Math.random() instead of UUID.randomUUID()
                 causing collisions under concurrent load
  Resolution Time: 6 hours
  Reported By:   Automated Regression Suite
  Reported Date: 2025-10-11
  Resolved Date: 2025-10-11
  Sprint:        Sprint 10
  Status:        CLOSED
  Description:   Transaction ID generation function used Math.random() seeded with
                 timestamp millis. Under concurrent execution, two threads generated
                 identical IDs, causing unique constraint violation on DB and silent
                 transaction drop.
  Fix Applied:   Replace Math.random() with java.util.UUID.randomUUID(); added DB-level
                 unique constraint with retry logic
  Regression Tests Added: Yes (concurrency test with 500 parallel threads)
